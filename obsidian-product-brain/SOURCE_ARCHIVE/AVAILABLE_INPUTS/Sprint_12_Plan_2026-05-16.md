## 15.1 Current Code Reality Summary

### Lesson route
`app/lesson/[id].tsx` ‚Äî stateful, section-by-section. Drives a `sec` index (-1 = chunk selector, 0-10 = section, 11+ = complete) through hardcoded 11-section switch. 11 imports at top of file, no component registry. Switch in `renderSection()` maps sec index to component. Per-section progress writes to storage via `mk({lessonId}-{sectionKey}, true)`. Mastery thresholds (60-70%) gating "Try Again" suggestion but never block "Continue Anyway".

### Lesson Zero
`app/lesson-zero.tsx` ‚Äî already a separate route (not under `/lesson/[id]`). Flag: `lm7_seen_lesson_zero`. Confirms the precedent for parallel routes.

### Sections architecture
`constants/sections.ts` exports `SECS` (11 keys) + `MASTERY_THRESHOLDS` + `CHUNKS` (3-part: Learn[0-3] / Practice[4-6] / Produce[7-10]). Adding a v1 9-section variant would mean a parallel constant (e.g. `SECS_V1`) ‚Äî no schema migration needed for legacy.

### 11 Section components in `components/sections/`
All follow a uniform contract: `props = { data + onComplete(score, total) + onError(word, section, given, correct) + say(text) }`. No persistence inside components. AI gated by `FEATURES.aiLesson` (SayItYourWay + MiniConversation return `null` if false). CombineWeave (17.9 KB) is the largest; Review (11.8 KB) second. Total ~75 KB section code.

### Lesson data shape (`lib/types.ts`)
`Lesson` interface = 22 fields (metadata + 11 typed arrays for each section's data + 6 enrichment fields + summary). **Not section-keyed** ‚Äî sections rely on parallel arrays (lesson.fillFG, lesson.sayIt, lesson.miniConv etc.). 24 legacy lessons in `LESSONS` registry. No section-level type enforcement; runtime renders whatever array is referenced.

### Pool shape
`FillItem` + `WeaveFillItem` (latter requires `fr` for TTS). 5 pool files (L1-L5), each ~270 items. Validator at `scripts/validatePools.ts`. Pools are static .ts arrays; no runtime schema validation.

### Practice tab
`app/(tabs)/practice.tsx` ‚Äî 4 modes: `menu | scenario | translate | lesson`. SRS via `useSRS()` hook (Leitner 5-box, storage key `lm7_srs`, intervals [0,1,3,7,30]). Data sources: `SCENARIOS` (data/practiceScenarios.ts) + `FLASH` (data/flashcards.ts) + pools (typed Weave for L1-L5). User-facing modes: Scenarios / Translate / Lesson Practice / Error Correction (if weak spots).

### Daily Review
`components/DailyReviewOverlay.tsx` ‚Äî modal MCQ quiz, 5 questions, weak-spot prioritized. Generated via `genReviewItems(count, weakSpots, maxEligibleLesson)`. **Simple format** ‚Äî NOT the 4-screen ritual (Fragments/Reading/Recognition/Outro) from V4-B canon. NOT the 4-step rhythm (Recall/Fill ‚Üí Micro-Contrast ‚Üí A Small Moment ‚Üí Mini Weave) from v1 ¬ß20. 6-exercise-type CR.A1 rotation is **aspirational, not implemented**.

### AI lesson behavior
`lib/ai.ts` ‚Äî thin Supabase Edge Function wrapper. 3 entry points: `evaluateSayIt`, `sendMiniConv`, `sendAIMessage`. 15-sec timeout, fallback strings on error. System prompt: A1-A2 French conversation partner, no direct error correction, max 1-3 sentences, no emojis. Edge functions: `ai-evaluate`, `ai-chat`, `ai-diag`. Provider abstraction lives in Edge Function layer (Gemini/Claude routing per CLAUDE.md, invisible to app code).

### Feature flags
`config/productStage.ts` ‚Äî `PRODUCT_STAGE = "sandbox" | "dev-apk" | "public-beta"` via `EXPO_PUBLIC_PRODUCT_STAGE` env. Default "sandbox" (everything on). Dev APK: paywall/aiChat/wordGraph/monLexique/leCarnet OFF; aiLesson/practice ON. `DEV_APK_LESSON_LIMIT = 5` (frozen Sprint 11). FEATURES object has 8 boolean flags + lesson limit.

### Storage

Separate keys: `lm7_srs`, `lm7_seen_lesson_zero`, `lm7_first_use_date`, `lm7_seen_how_weave_works` (added by WS.6 `3c9d9ef`). AppProvider context loads on app start, syncs to Supabase (gated by auth).

### Renderer strategy

```tsx
// components/lesson-v1/LessonRendererV1.tsx
export function LessonRendererV1({ lesson }: { lesson: Lesson }) {
  const [i, setI] = useState(0);
  const screen = lesson.screens[i];
  const onComplete = () => setI((n) => n + 1);

  switch (screen.type) {
    case "meet-card":      return <MeetCard payload={screen.payload as MeetPayload} onComplete={onComplete} />;
    case "insight-card":   return <InsightCard payload={screen.payload as InsightPayload} onComplete={onComplete} />;
    case "fill-with-traps":return <FillWithTraps payload={screen.payload as FillPayload} onComplete={onComplete} />;
    case "weave":          return <Weave payload={screen.payload as WeavePayload} onComplete={onComplete} />;
    case "say-it-your-way":return <SayItYourWayV1 payload={screen.payload as SayItPayload} onComplete={onComplete} />;
    case "natural-reveal": return <NaturalReveal payload={screen.payload as NaturalRevealPayload} onComplete={onComplete} />;
    case "recap":          return <RecapCard payload={screen.payload as RecapPayload} onComplete={onLessonDone} />;
    default: return null;

}
```
