> **PILOT.** Third **Architecture Verb** opening (after L2 être, L7 aller, L9 faire), built as a **split-sense help/permission opening** (archetype #2). It opens **`pouvoir`** on **one narrow slice — asking for help, asking permission, and saying whether you can do a *small known action*** — and **holds back** pouvoir's broad senses (possibility/probability `il se peut`/`peut-être`, the conditionnel `je pourrais`, formal `puis-je`, the full paradigm). It is the **first new engine after the L10 integration beat**, the first spec from the L10–L20 band map (Option C), and it **lands the `vous pouvez m'aider ?` hook L10 seeded** while reusing L9 (`faire ça`/`faire une pause`) and the L10 after-class scene. §18 records the findings.

> **Archetype**: primary = **Architecture Verb** (#2), as a **split-sense help/permission opening**. Secondary (flavor only, not a second budget) = **Chunk / Natural Speech / Human Expression** (#1) + **Review / Integration** flavor (it recombines L9–L10). **New grammar systems: 0. Full `pouvoir` ownership: 0.**

## Canon Alignment Note (read first)

Spine: **L0 → … → L9 → L10 (integration) → L11 (this)**. L10 left an after-class help scene that *wants* an ability/permission verb and explicitly **seeded `pouvoir`** via the recycled frozen `pouvez-vous répéter ?` + the recognition hook `vous pouvez m'aider ?`. L11 cashes that hook on `pouvoir`'s most human, concrete slice — **can I…? / can you help? / I can't** — reusing the owned `verb + infinitive` mechanic (L6 `je voudrais + inf.`) and `ne…pas` (L3) over **already-owned actions** (L9 `faire ça`/`faire une pause`, L7 `aller`).

**Split-sense opening (third instance of the pattern).** Per archetype #2's "split-sense opening — repeatable pattern": `être`/`avoir` were opened broadly; `aller` (movement) and `faire` (small-action) were opened narrowly. **L11 opens `pouvoir` narrowly — help/permission + can-I-do-this-known-action only** — and **defers** the broad senses: possibility/probability (`il se peut (que)`, `peut-être`), the **conditionnel `je pourrais`** ("I could", softer), formal `puis-je`, and the full paradigm. This protects pedagogy (prerequisite-safety) and **commercial depth** — paywall = **Campfire ~L24** (band map), so L11 is **pre-Campfire / free**: useful, but it does **not** spend pouvoir's deeper modal value.

**What L11 is *not*:**
- **not** a full `pouvoir` architecture-verb lesson (one dominant active core — `je peux` — the rest supported/recognition);
- **not** a full modal-verb grammar lesson (no can/may/be-able taxonomy);
- **not** a full permission-system lesson;
- **not** a conditionnel / `pourrais` lesson (`je pourrais` = recognition/deferred);
- **not** a subjunctive doorway (`il se peut que …`, `il faut que …` = deferred);
- **not** a question-formation lesson (`est-ce que je peux …?` = recognition; asking uses **rising intonation**, e.g. `je peux faire une pause ?`);
- **not** an object-pronoun lesson (`m'` in `vous pouvez m'aider ?` is in-chunk only; the object-pronoun *system* stays deferred).

**Prerequisite-safety notes (explicit):**
- After `pouvoir`, the next verb stays **infinitive**, and those infinitives are **already owned** (`faire` L9, `aller` L7, `comprendre` L6, `répéter` in the rescue chunk). So the only genuinely new lexis is `pouvoir` itself (+ `aider` in-chunk).
- `vous pouvez m'aider ?` / `vous pouvez répéter ?` are **fixed help chunks** (like `pouvez-vous répéter ?`); `aider` and the object pronoun `m'` enter **only inside them**.

**Café-centricity:** café does not appear; the scene is **help / rest after class** (human), continuing the L6→L10 reduction.

**Legacy mapping divergence (flag).** `CLAUDE.md`'s legacy 24-lesson list maps **L11 = Everyday Structures**. The **v1 syllabus spine** maps **L11 = Pouvoir-light / Help & Permission** (per the band map). Per the Master Pipeline precedence rule (*newer active canon > codebase canon > older active canon*), this spec follows the **v1 spine**; the legacy bracket is noted, not adopted. Runtime/dev-apk (L1–L5 only) is unaffected — planning canon ahead of runtime.

## 3. Engine Plan

> One dominant new full-cycle engine (**permission: can I…?**), two short/supported satellites (**help via `vous pouvez`**, **can't**), recognition-only flavor for the deferred senses. Architecture-verb guardrail: **one active core (`je peux`), the rest supported/recognition.**

| Engine | Depth | Why it exists |
|---|---|---|
| **A — Permission / can I? (`je peux + known action`)** | **full-cycle** (the one new engine) | The lesson's main target: **ask if you can** do a small known action. `frame:je-peux-plus-infinitive` + `frame:je-peux-faire-plus-action`, filled with **owned** actions. Examples: *Je peux faire ça ? · Je peux faire une pause ?* (supported, optional: *Je peux aller à la maison ?*). Kept as **help/permission**, not broad ability. |
| **B — Help with `vous pouvez`** | short-cycle / supported | Reuse the L10 hook + rescue to **ask for help**: `frame:vous-pouvez-plus-infinitive`. Examples: *Vous pouvez m'aider ? · Vous pouvez répéter ?* (+ recycled formal *Pouvez-vous répéter ?*). **Not** a full vous-form conjugation — useful help chunks; object pronoun `m'` in-chunk only. |
| **C — Cannot / refusal (`je ne peux pas + known action`)** | short-cycle / supported | Reuse `ne…pas` + pouvoir to say **you can't** do a known action: `frame:je-ne-peux-pas-plus-infinitive`. Example: *Je ne peux pas faire ça.* **Do not** over-expand negative pouvoir; tie to known actions. |
| **Recognition — broad pouvoir + conditionnel + formal/question forms (future hook)** | **recognition only** | `on peut` / `tu peux` / `il peut` / `nous pouvons` / `ils peuvent` (paradigm); **`je pourrais` / `tu pourrais`** (conditionnel); `peut-être` / `il se peut` (possibility); `puis-je …?` (formal); `est-ce que je peux …?` (formal question — L12 bridge). **Met, never produced.** |

**Why this is not a full `pouvoir` lesson.** Only `je peux` carries the active core; `vous pouvez` (help) + `je ne peux pas` are supported; everything else — conditionnel, possibility, formal, the full paradigm — is recognition/deferred (§2). The infinitives are **owned**, so the only new lexis is `pouvoir`. The teaching surface is **a help/permission human scene + three frames**, never a conjugation grid or a modal-logic lecture.

**Why this follows L10 naturally.** L10 seeded `vous pouvez m'aider ?` and left the actions (`faire ça`, `faire une pause`) waiting to be modalized. L11 adds one modal and lets the learner ask permission, ask for help, and decline — using those exact actions. A small step, not a new system.

**Why this does not become broad modal grammar.** `pouvoir` as possibility/probability (`il se peut`), as conditionnel softening (`je pourrais`), and the can/may/be-able taxonomy are all recognition/deferred. The owned sense is concrete help/permission. `trap:pouvoir-broad-possibility-too-early` and `trap:pouvoir-conditionnel-too-early` guard production.

## 4. Opening Sentence Family

> **Human after-class / help scene.** *"You're tired after class. You need help, you want a pause, and you need to ask if you can do something — or if someone can help you."*

| Role | Sentence | Note |
|---|---|---|
| **Open** | `Bonjour, je suis fatigué.` | recycled (L1/L6) |
| **Need (recycled)** | `J'ai besoin d'aide.` | L6 (recycled) |
| **Ask for help (anchor)** | `Vous pouvez m'aider ?` | **signature**: L10 hook landed (supported fixed chunk) |
| **Ask to repeat** | `Je ne comprends pas. Vous pouvez répéter ?` | rescue via `vous pouvez + inf.` (supported); recycled `pouvez-vous répéter ?` also fine |
| **Want a pause (recycled)** | `Je voudrais faire une pause.` | L9 (recycled) |
| **Ask permission** | `Je peux faire une pause ?` | **new core**: `je peux + faire une pause` (rising intonation) |
| **Can't (refuse)** | `Je ne peux pas faire ça.` | **new core**: `je ne peux pas + faire ça` |
| **Movement (recycled)** | `Je vais à la maison.` | L7 (recycled) |
| **Reassure (recycled)** | `C'est pas grave.` | L2 chunk |
| **Close** | `Merci beaucoup.` | L1 polite close |

- **Interchangeable pieces**: `je peux { faire ça / faire une pause }` · `je ne peux pas { faire ça }` · `vous pouvez { m'aider / répéter } ?` (fixed help chunks) · *(supported, optional)* `je peux aller à { la maison } ?`.
- **Fixed frames**: `frame:je-peux-plus-infinitive` · `frame:je-peux-faire-plus-action` · `frame:je-ne-peux-pas-plus-infinitive` · `frame:vous-pouvez-plus-infinitive`. *(Recognition only)* `frame:est-ce-que-je-peux-plus-infinitive`.
- **Replaceable slots**: the **infinitive/action** after `peux` — **closed owned set** (`faire ça`, `faire une pause`, `aller à + place` (supported), `comprendre`); `vous pouvez` takes `m'aider` / `répéter` (fixed).
- **Anchor**: `Vous pouvez m'aider ?` · **Variations**: `Je peux faire une pause ?`, `Je ne peux pas faire ça.`, `Vous pouvez répéter ?` · **Contrast**: `Je peux faire ça ?` (permission, intonation) vs `Est-ce que je peux faire ça ?` (formal question — **recognition only**, shown to seed L12) and vs `Je pourrais faire ça.` ("I could" — **recognition only**, the conditionnel softer form, shown but not owned).
- **Rescue / natural sentence**: `Je ne comprends pas. Vous pouvez répéter ?` / `Pouvez-vous répéter ?` + `c'est pas grave`.
- **Forbidden / not-yet-ready substitutions**:
  - **no conditionnel `pourrais`** (`je pourrais …` = recognition only);
  - **no possibility/probability** (`il se peut`, `peut-être` = recognition only);
  - **no formal `puis-je …?`** (recognition only);
  - **no `est-ce que` production** (`est-ce que je peux …?` = recognition; ask by intonation);
  - **no full-paradigm production** (`on peut`, `tu peux`, `il peut`, `nous pouvons`, `ils peuvent` stay recognition);
  - **no object-pronoun system** beyond the fixed `m'aider` chunk;
  - **no new infinitive lexis** after `pouvoir` (closed owned set; no long modal chains);
  - **no present `vouloir`** (`je veux`); **no futur proche production**; **no past/future**.

> **Is L11 too much after L10, or well-timed?** **Well-timed, with a medium watch (band map's flag).** The genuinely new load is **one modal verb + the permission/help mechanic** — infinitives, `ne…pas`, and the help chunks are owned/recycled. Total exposure runs a touch high **only because many pouvoir forms are parked at recognition** (paradigm + conditionnel + possibility); **production stays at 6**. Watch-items: (a) the `m'aider` object pronoun (contained to one chunk), (b) **conditionnel `pourrais` creep** (the softer "could" is a strong leak), (c) register spread (`je peux ?` intonation vs `est-ce que`/`puis-je`/`pourrais` recognition). If heavy in smoke, demote `je peux aller` and trim recognition; **do not** promote conditionnel/possibility/paradigm.

## 10. French-Specific Contrast / Transfer Trap

> Micro-contrasts only (template §7.10) — "Why This Works" / Natural Reveal fuel. **Keep ability/permission as usage, not a lecture.**

- **"Can I do that?" → `Je peux faire ça ?`** After `pouvoir`, the verb stays **infinitive** (`faire`), **no `à`**, and you ask by **rising intonation**.
- **"Can you help me?" → `Vous pouvez m'aider ?`** `pouvoir + infinitive` directly (no "to"); `m'` ("me") learned **whole**.
- **"I can't do that" → `Je ne peux pas faire ça.`** `ne…pas` wraps `peux`; `faire ça` stays as the known action (not `je ne peux pas ça`).
- **Do not overexplain the "can / may / be able to" taxonomy** — show the natural French; the conditionnel `pourrais` ("could") and possibility `il se peut` come later.

## 12. Natural Reveal / Feedback Plan

**Ask for help (signature):**
- **Expected**: `Vous pouvez m'aider ?`
- **Acceptable alternatives**: `Pouvez-vous m'aider ?` (formal); `J'ai besoin d'aide.` (owned, L6); with `s'il vous plaît`.
- **Natural upgrade**: add `s'il vous plaît`.
- **Common mistakes**: **`vous pouvez aide`** / **`je peux aide`** (noun `aide` instead of infinitive `aider`); dropping `m'`. **Hint**: "It's the verb: `vous pouvez **m'aider**`."
- **Weak-point tags**: `weak:object-pronoun-in-chunk`, `weak:pouvoir-paradigm`.
- **Passive mirror**: "You asked for help."

**Ask permission:**
- **Expected**: `Je peux faire une pause ?` / `Je peux faire ça ?`
- **Common mistakes**: `je peux fais` (conjugated after modal); `je suis peux` (être + pouvoir); `je peux à faire`. **Hint**: "`je peux` + the plain verb: `je peux **faire**`."
- **Weak-point**: `weak:peux-plus-infinitive`.

**Can't:**
- **Expected**: `Je ne peux pas faire ça.`
- **Acceptable alternatives**: spoken `je peux pas faire ça` (recognised, lightly upgraded).
- **Common mistakes**: `je ne peux pas ça` (dropping the verb); misplacing `ne…pas`. **Hint**: "Keep the action: `je ne peux pas **faire** ça`."
- **Weak-point**: `weak:negation-placement`.

**Recognition forms (feedback is recognition-framed, never a production correction):**
- **Shown**: `Est-ce que je peux …?` / `Je pourrais …` / `Il se peut.` — "These are more formal or softer ways; you'll meet them later. For now, `je peux … ?` and `vous pouvez … ?` are perfect."
- **If the learner *attempts* `pourrais`/`est-ce que`/`puis-je`/possibility/full paradigm**: accept the meaning, mirror gently, tag `weak:pouvoir-overbroad`, defer — **do not** teach on the spot (contract §11).

**Common mistakes (lesson-wide watch list):** `je peux aide` / `vous pouvez aide` · `je peux fais` · `je ne peux pas ça` · **`pouvais`/`pourrais`/`peux` confusion** (don't reach for past `pouvais` or conditionnel `pourrais`) · `est-ce que` overload · broad possibility (`il se peut`) too early · full-paradigm slips. **Passive mirror throughout; no scores, no reward language.**

The help/permission slice is `pouvoir`'s most human, useful part and fits the after-class scene.

**Is L11 better here than earlier — or should it have come before faire?** **Here is right.** `pouvoir` is most useful when there are **actions to modalize**; L9 (faire) and L7 (aller) provide them, so `je peux faire ça` / `je peux faire une pause` are owned-action recombinations. Opening `pouvoir` **before** faire would have left it with little concrete to attach to. So **faire (L9) → pouvoir (L11)** is the correct order, with the L10 integration beat between.

**Should L12 be question expansion 1, as the band map suggests?** **Leaning yes — L11 builds the bridge.** `est-ce que je peux …?` recognition is a natural lead-in to **owning `est-ce que`** at L12 (with L8 `où`). But the band map's L12 gate is still open; **re-map before writing L12**. The can/can't engine also **pairs with `devoir`/`il faut`** (ability ↔ obligation) at the L15 candidate.

**Does L11 reveal any required patch to template / archetype / AI contract / ID convention?** Minor, **non-blocking** (propose for review, do **not** auto-apply):
- **AI Generation Contract v1 §15** should gain the **L11 row** proposed in §16.

Worth a one-line note (template/archetype) that **object pronouns, like question words, may enter as frozen-chunk pieces before their system is owned.**
- **Archetype #2** — `pouvoir` is the **third** split-sense instance; the "repeatable pattern" note already covers it (could cite pouvoir as a third example).

**Does L11 preserve commercial depth before Campfire ~L24?** **Yes.** It owns only the **help/permission** slice; the genuinely deep, broadly-applicable pouvoir — conditionnel softening (`je pourrais`), possibility/probability (`il se peut que` + subjonctif), the full paradigm, and the polite-softening-of-everything register — is **reserved** for later/paid lessons. L11 gives strong free usefulness without spending pouvoir's deeper value.

**Does L11 provide enough material for the AI Generation Contract?** **Yes — a strong fifth fixture.** L6 "don't add a word"; L7 "don't promote recognition to production"; L8 "don't escalate the question register"; L9 "don't let a huge verb spill past its sense"; **L11 = "don't open a modal's conditionnel/possibility senses, and don't let an in-chunk object pronoun (`m'`) become a productive system."**
