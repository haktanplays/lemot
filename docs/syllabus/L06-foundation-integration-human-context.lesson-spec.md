It executes the retrospective's top recommendation: a consolidation beat that **calms the L2–L3 difficulty spike**, **reduces café/transactional centricity**, and **recombines L1–L5 engines in human contexts** — with **zero new grammar systems**. §18 records the findings.

## Canon Alignment Note (read first)

Spine: **L0 → L1 → L2 → L3 → L4 → L5 → L6 (this)**. L6 is the **integration lesson the L1–L5 retrospective called for** (retrospective §14/§16: relieve the L2–L3 spike + break café-centricity before opening the time dimension at L7).

**What L6 is *not*:** not a new major grammar lesson, not an architecture-verb lesson, not a tense/aspect/mood doorway. It introduces **no new system**. Its only genuinely new lexis is **`word:aide`** (help) and the promotion of **`word:comprendre`** from a frozen rescue chunk into infinitive use (`je voudrais comprendre`) — and `comprendre` is **not** newly conjugated.

**Café-centricity fix (explicit):** the retrospective flagged café recurring in L0/L1/L4/L5. L6's sentence family is a **classroom / small-group help scene** with **no café in the family at all**; café appears only once, in a single production target that pairs it *against* a non-café state (`j'ai besoin d'un café, mais je n'ai pas faim`). The dominant world shifts to **human needs, states, identity, and asking for help.**

## 3. Engine Plan

> **No new engine is introduced.** The "primary engine" is **recombination of owned engines** — the integration skill itself. Depth labels describe how much *production weight* each recombination cluster carries.

| Engine | Depth | Why it exists |
|---|---|---|
| **A — Recombination of known engines** | **full-cycle** (recombination, not new) | Re-fire L1–L5 frames together: `je suis ± state/identity`, `je ne suis pas + state`, `j'ai besoin de + noun`, `je voudrais + infinitive/noun`, `je n'ai pas de + noun`. The lesson's core skill is *connecting* them. |
| **B — Human-context broadening** | short-cycle | Move the scene from café/order to **human need / social interaction**: `je suis fatigué`, `j'ai besoin d'aide`, `je ne comprends pas`. Introduces `word:aide`. |
| **C — Repair / natural response** | short-cycle | Reuse L1 rescue + L3 negation naturally in a real exchange: `je ne comprends pas`, `pouvez-vous répéter ?`, `c'est pas grave`. |
| **Ambient — movement / next step (future hook)** | ambient (recognition) | `je voudrais aller …` / `on y va` are *met* (reusing the known `je voudrais + infinitive` frame) but **not** taught — a tiny preview of L7's `aller` doorway. |

**Why this is a review/integration lesson, not a new-grammar lesson:** every frame is already owned; the only new lexis is `aide` + `comprendre`-infinitive; no tense, no conjugation, no new system. **Why it breaks café-centricity:** the sentence family is a classroom/help scene; café appears once, contrastively.

## 4. Opening Sentence Family

> **Human scene, not a café order.** *"You're in a class / small group. You're tired, not ready, and you need help — but you can still explain yourself politely in French."*

| Role | Sentence | Note |
|---|---|---|
| **Open** | `Bonjour.` | greeting, no order |
| **State (être)** | `Je suis fatigué.` | L2 state, human context |
| **Negated state (être + ne…pas)** | `Je ne suis pas prêt.` | L2 `prêt` + L3 negation recombination |
| **Need (avoir + new noun)** | `J'ai besoin d'aide.` | signature L6: `j'ai besoin de` + `word:aide` |
| **Want + infinitive (promotion)** | `Je voudrais comprendre.` | `je voudrais` + `comprendre` infinitive |
| **Rescue** | `Je ne comprends pas. Pouvez-vous répéter ?` | L1 rescue, in context |
| **Reassure** | `C'est pas grave.` | L2 chunk, natural response |
| **Negated object** | `Je n'ai pas de table.` | L5 `pas de` recombination |
| **Identity contrast** | `Je suis étudiant, pas médecin.` | L2 identity + elliptical `pas` |
| **Close** | `Merci beaucoup.` | L1 polite close |

- **Interchangeable pieces**: `je suis ± { fatigué / prêt / étudiant }` · `j'ai besoin de { aide / un café }` · `je voudrais { comprendre / un café }` · `je n'ai pas de { table / café }`.
- **Fixed frames** (all recycled): `frame:je-suis-plus-state` · `frame:je-ne-suis-pas-plus-state` · `frame:j-ai-besoin-de-plus-noun` · `frame:je-voudrais-plus-infinitive` · `frame:je-n-ai-pas-plus-noun`.
- **Replaceable slots**: the state/identity after `je suis`; the noun after `besoin de`; the infinitive after `je voudrais`; the object after `pas de`.
- **Contrast sentence**: `J'ai besoin d'aide.` (correct) vs `Je suis besoin.` (transfer error — need takes avoir).
- **Forbidden / not-yet-ready substitutions**: no `aller`/futur proche production (`je voudrais aller` = recognition future-hook only); no new `comprendre` conjugation (`je comprends` only inside the frozen rescue chunk); no past/future; no vocabulary dump (only `aide` + `comprendre` are new); no new grammar system.

## 10. French-Specific Contrast / Transfer Trap

> Micro-contrasts only (template §7.10).

- **"I need help" → `j'ai besoin d'aide`, not `je suis besoin`.** Need = avoir (the L4 être↔avoir boundary, reapplied to a new noun).
- **"I'd like to understand" → `je voudrais comprendre`, not `je voudrais comprends`.** After `je voudrais`, the verb stays **infinitive**.
- **"I'm tired but I'm hungry" → `je suis fatigué, mais j'ai faim`.** Two systems (être-state, avoir-sensation) in one sentence — the clearest possible proof the learner controls the boundary.

## 12. Natural Reveal / Feedback Plan

**Need help (signature):**
- **Expected**: `J'ai besoin d'aide.`
- **Acceptable alternatives**: `J'ai besoin d'aide, s'il vous plaît.`; unaccented variants.
- **Natural upgrade**: add `s'il vous plaît` for warmth.
- **Common mistakes**: **`je suis besoin`** (être for a need); dropping `de` (`j'ai besoin aide`). **Hint**: "Need takes *avoir* + `de`: `j'ai besoin **d'**aide`."
- **Weak-point tags**: `etre-vs-avoir`, `de-before-noun`, `elision`.
- **Passive mirror**: "You asked for help, clearly and politely."

**Want to understand:**
- **Expected**: `Je voudrais comprendre.`
- **Common mistakes**: `je voudrais comprends` (conjugated). **Hint**: "After `je voudrais`, the next verb stays in its base form: `comprendre`."
- **Weak-point**: `infinitive-only`.

**Negated state / identity contrast:**
- **Expected**: `Je ne suis pas prêt.` / `Je suis étudiant, pas médecin.`
- **Common mistakes**: `je suis pas prêt` (spoken vs full — recognized, lightly upgraded). **Hint**: "Full form `je ne suis pas` is clearest while learning."
- **Weak-point**: `ne-placement`.

**Tired-but-hungry (boundary proof):**
- **Expected**: `Je suis fatigué, mais j'ai faim.`
- **Passive mirror**: "Two feelings, two verbs — each in its place."

**Rescue (recycle):** `Je ne comprends pas. Pouvez-vous répéter ?` — neutral, unchanged; now sits naturally beside `je voudrais comprendre`.

## 16. AI Generation Compatibility

L6 is integration-heavy, so generated content must be tightly bounded:

- **AI may recombine only L1–L6 allowed frames/items** (the `word:`/`chunk:`/`frame:` IDs marked active/supported across L1–L6).
- **AI may introduce at most 1–2 low-risk nouns**, and only if marked **supported/recognition** (e.g. a relatable concrete noun) — never a verb, never a new system.
- **AI must NOT introduce** new tense, new verb conjugation, `aller`, past tense, futur proche, or question-word systems — *even if the French is correct* (prerequisite-safety overrides validity).
- **AI must keep human-context broadening without vocabulary dumping** — recombine known pieces into new human scenes, don't pile on new words.
- **AI must follow Natural Reveal tone** (passive mirror, neutral on mistakes) and **respect active/supported/recognition boundaries** per item.

> This section + §9 of the ID convention are the **direct seed for `docs/syllabus/ai-generation-contract-v1.md`** (retrospective §12/§15). L6 is a good stress case for that contract because integration lessons are where a generator is most tempted to "add just one more useful word."

## 18. L6 Pilot Findings

**Does the Review / Integration archetype work?** **Yes — and it earns its place.** L6 proves the foundation's engines *connect*: the learner produces 8 human-context lines using five different prior engines with only two new words. The archetype's value is exactly what the retrospective predicted — consolidation + context-broadening with near-zero new load.

**Does L6 confirm the need for integration after L1–L5?** **Yes.** Building it surfaced how café-bound the prior material was (every natural recombination wanted to reach for `café`; the spec had to *push* toward aide/help/classroom). That friction is the evidence the integration beat was needed. It also confirms the L2–L3 spike is real: L6's calm (4 active-new) only feels calm *by contrast* with those two.

**Should L7 be Aller / futur proche or another integration step?** **Aller / futur proche.** L6 does the consolidation; a *second* integration lesson now would feel like a test (archetype #10 risk) and stall momentum. L6 already plants the hook (`je voudrais aller` recognition). L7 should open the **time dimension** — it's the foundation's biggest gap (retrospective §9) and reuses the infinitive-chain mechanic L6 just promoted.

**Does L6 reveal any required patch to template/archetype/ID convention?** Minor, **non-blocking** (do **not** auto-apply — propose for review):
- **Archetype doc #10 (Review/Integration)** could gain a note from this pilot: *"An integration lesson may introduce 1–2 low-risk new words (e.g. a human-need noun) to give a fresh hook, but 0 new systems; lean on Weave/Say-It/Natural-Reveal over explanation; frame as 'you can now express yourself,' not a quiz, to avoid the test-feel."*
- **ID convention**: L6 (first authored in v0.1) used it cleanly; no change needed. It validated that recombination chunks (`chunk:je-suis-fatigue-mais-j-ai-faim`) get sensible IDs.

**Does L6 provide enough material for a future AI Generation Contract v1?** **Yes — it's the ideal stress case.** §16 + ID-convention §9 give a generator clear allowed-ID boundaries and the "no new system even if valid" rule. An integration lesson is precisely where a generator is most tempted to add vocabulary, so the L6 constraints should be a core fixture of `ai-generation-contract-v1.md`.
