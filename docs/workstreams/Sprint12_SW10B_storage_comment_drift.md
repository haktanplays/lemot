## Tier

## Problem statement
SW10 Codex review confirmed M1: documentation and code comments around storage refer to the wrong storage backend.

- `lemot-app/hooks/useStorage.ts` comments at lines 17 and 34 say "AsyncStorage" but the actual call is to `kvStorage` from `@/lib/storage`.
- `lemot-app/lib/storage.ts` imports from `expo-sqlite/kv-store` but binds the local symbol as `AsyncStorage`, which is misleading for any reader expecting the deprecated `@react-native-async-storage/async-storage` package.
- Root `CLAUDE.md` line 50 declares `Storage: MMKV (Expo app)`. MMKV is not used. The actual native backend is `expo-sqlite/kv-store`; the actual web backend is `window.localStorage`.
- `react-native-mmkv` is in `package.json` but unused by `lib/storage.ts`. **This PR does NOT touch dependencies.**

The drift is cosmetic but actively misleads. Code reviewers, future Claude sessions, and operator memory all suffer.
